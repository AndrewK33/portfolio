Вариант использования Подтверждение номера телефона по средству ввода смс-кода. Отправка смс-кода должна производиться с помощью сервиса SMSС
ID 6
Краткое описание Подтверждение номера телефона по средству ввода смс-кода. Отправка смс-кода должна производиться с помощью сервиса SMSС
Актор(ы) "Пользователь", "Система", "Сервис SMSC"
Предварительные условия -
Действия 1. ВИ начинается, когда "Пользователь" завершает заполнение данных
2. "Система" выводит сервисное сообщение: "Для продолжении процедуры "Регистрации" подтвердите Ваш номер телефона, путем введение в "Необходимое поле" одноразового пароля из СМС"
2.1. "Сестема" обращается к сервису "Генератор одноразовых паролей"
2.2. Сервис "Генератор одноразовых паролей" отправляет результат в сервис "SMSC", с копией "Системе"
2.3. Сервис "SMSC" направяет одноразовый пароль по номеру телефона, в соответствии с данными "Номер телефона" заполненными в поле "Å"
3. "Пользователю" приходит уведомление с одноразовым паролем
4. "Пользователь" заполняет одноразовый пароль в "Необходимое поле"
5. "Система" верифицирует корректность введенных данных
5.1 "Система" автоматически направляет пользователя к следующему шагу процедуры "Авторизации" в приложения
5.2. "Система" выводит сервисное сообщение о некорректности введеного номера.
В этом случае процедуры 2.1 - 2.3 повторяются
6. "Пользователь" переходит к следующему шагу процедуры "Авторизация"
Результат "Пользователь" переходит к следующему шагу процедуры "Авторизация"
Ограничения "Пользователь" не может ввести номер телефона другого "Пользователя" для отправки на него одноразового пароля
"Пользователь" не может  перейти к "Ленте" приложения если данные введены некорректно
Ввести ограничение по времени 0.59 секунд на повторную отправку одноразового пароля
Дополнительные атрибуты
 1. Алгоритм создания одноразовых паролей должен соответствовать стандартам:
RFC 1760 (S/Key), RFC 2289 (OTP), RFC 4226 (HOTP) и RFC 6238 (TOTP)
2. Необходимо предусмотреть функцию автозаполнения одноразового пароля из уведомления (Autofill bar)
