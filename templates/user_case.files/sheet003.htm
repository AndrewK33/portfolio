Вариант использования Возможность прочтения и принятия "Пользовательского соглашения"
ID 2
Краткое описание Ознакомление с соглашением
Актор(ы) "Пользователь", "Система"
Предварительные условия Первый запуск приложения/ было внесено изменение в "Пользовательское соглашение"
Действия 1. ВИ начинается, когда "Пользователь" включает приложение
2. "Система" выводит "Приветственное послание", в котором "Пользователю" предлагают прочитать "Пользовательское соглашение" или  прочитать обновление в "Пользовательском соглашении" (в случае наличия)
2.1. "Система" выводит предложение воспользоваться функцией "Скипнуть прочтение" с примечанием, что в данном случае соглашение с условиями "Пользовательского соглашения" будет произведено автоматически
2.1.1 "Пользователь" выбирает  функцию "Скипнуть прочтение"
2.1.2 "Система" выводит уведомление об автоматическом соглашении с условиями "Пользовательского соглашения" и направляет на экран предлагающий пройти процедуру "Авторизации"
3. "Пользователь" ознакамливается/скролит текст "Пользовательского соглашения"
4. По итогу прочтения текста "Пользовательского соглашения" "Пользователь" принимает решение подтвердить согласие, выделением чек-бокса у фразы, предлагающей согласиться с условиями "Пользовательского соглашения"
4.1. По итогу прочтения текста "Пользовательского соглашения", "Пользователь" принимает решение не соглашаться с условиями "Пользовательского соглашения" и прекращает работу с приложением
5. "Система" делает кнопку "Продолжить" активной
6. "Пользователь" нажимает на кнопку "Продолжить" и его направляет на экран предлагающий пройти процедуру "Авторизации"		  
Результат 1. "Пользователь" согласился с условиями "Пользовательского соглашения" и нажимает кнопку "Продолжить"
2. "Пользователь" прекратил использование приложения
Ограничения Если "Пользователь" не воспользовался функцией "Скипнуть прочтение" и приступил к прочтению "Пользовательского соглашения", то без выделения чек-бокса у фразы, предлагающей согласиться с условиями "Пользовательского соглашения", "Пользователь" не сможет продолжить дальнейшую работу в приложении
Дополнительные атрибуты
 1. Язык отображаемого текста  в приложении должен соответствовать  настроенной на устройстве (телефон, планшет, иное) языковой политике
2. Текст должен быть читаем, хорошо контрастировать с фоном
3. Должна быть предусмотрена функция увеличения и выделения текста
4. Необходимо предусмотреть паттерн для людей людей с ограниченными возможностями для принятия согласия с условиями "Пользовательского соглашения" помимо выделения чек-бокса (voice-assistans и т.д.)
5. Кнопка "Продолжить" априори не активна, под кнопкой красным цветом выделено сервисное сообщение, которое уведомляет "Пользователя" о том, что: "Пользователь" не подвердил согласие с условиями "Пользовательского соглашения" и не может продолжать работу с приложением без подтверждения согласия
Предложения для обсуждения В случае если "Пользователя" не устраивает текст "Пользовательского соглашения" необходимо дать возможность "Пользователю" написания предложения для корректировки текста "Пользовательского соглашения", необходимо предусмотреть специальную форму для данного действия.
Примечание:
Каждый пользователь отказавшийся использовать приложение это потеря потенциальной прибыли для компании
Нужно учитывать особенности (физологические и ментальные) каждого пользователя
