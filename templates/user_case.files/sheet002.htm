Вариант использования "Авторизация" "Пользователя" в приложении
ID 1
Краткое описание Поэтапная процедура "Авторизация" "Пользователя" в приложении
Актор(ы) "Пользователь", "Система"
Предварительные условия Запуск приложения, подтверждение согласия с "Пользовательским соглашением"
Действия 1. ВИ начинается, когда "Пользователь" успешно выполнил "Функцию 1"
2. "Система" переводит "Пользователя" на экран для заполнения полей "Логин" и "Пароль". Внизу имеется 2 дополнительных кнопки "Забыли пароль?" и "Регистрация"
2.1 "Пользователь" заполняет поля "Логин" и "Пароль"
2.2. В случае успешного заполнения полей выполняется "Функция 8"
3. "Пользователь" нажимает на кнопку "Регистрация"
4. "Система" показывает скрытые поля (Меню расширяется) для заполнения необходимых данных  "Номер телефона", "Пароль", "Почта"....
5. "Пользователь заполняет поля, используя в том числе "Функцию 3" и "Функцию 4"
6. "Система" исполняет "Функцию 5", после которой "Пользователя" обратно переводит на экран для заполнения полей "Логин" и "Пароль"
7. "Система" исполет "Функцию 7"
Результат "Пользователь" завершил "Авторизацию" и перешел к "Ленте" приложения
Ограничения 1. Не выполнение "Пользователем" условий "Функции 2"
2. Наличие критериев запрещающих "Авторизацию" "Пользователя", "Функция 6"
Дополнительные атрибуты
 Поле "Номер телефона" = "Логин"
